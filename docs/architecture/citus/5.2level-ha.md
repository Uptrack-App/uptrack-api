

⏺ Citus HA Support

  Yes, Citus supports HA at two levels:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                         CITUS HA ARCHITECTURE                          │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  1. COORDINATOR HA → Patroni (streaming replication)                   │
  │  2. WORKER HA → Shard Replication (built into Citus)                   │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘

  ---
  1. Coordinator HA (via Patroni)

  Coordinator is just PostgreSQL, so standard Patroni works:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                      COORDINATOR HA                                     │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │         ┌─────────────────┐       ┌─────────────────┐                  │
  │         │  Coordinator    │       │  Coordinator    │                  │
  │         │  (PRIMARY)      │ ───►  │  (STANDBY)      │                  │
  │         │  netcup-1       │ sync  │  netcup-2       │                  │
  │         └────────┬────────┘       └────────┬────────┘                  │
  │                  │                         │                            │
  │                  └──────────┬──────────────┘                           │
  │                             │                                           │
  │                        ┌────▼────┐                                     │
  │                        │  etcd   │  (leader election)                  │
  │                        └─────────┘                                     │
  │                                                                         │
  │  If PRIMARY dies → Patroni promotes STANDBY → App reconnects           │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘

  ---
  2. Worker HA (Shard Replication)

  Citus can replicate each shard across multiple workers:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                      WORKER HA (Shard Replication)                      │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  citus.shard_replication_factor = 2                                    │
  │                                                                         │
  │  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐                 │
  │  │   Worker 1   │  │   Worker 2   │  │   Worker 3   │                 │
  │  │              │  │              │  │              │                 │
  │  │  Shard 1 ◄───┼──┼── Shard 1   │  │              │                 │
  │  │  (primary)   │  │  (replica)   │  │              │                 │
  │  │              │  │              │  │              │                 │
  │  │  Shard 2     │  │  Shard 2 ◄──┼──┼── Shard 2   │                 │
  │  │  (replica)   │  │  (primary)   │  │  (replica)   │                 │
  │  │              │  │              │  │              │                 │
  │  │  Shard 3 ◄───┼──┼──────────────┼──┼── Shard 3   │                 │
  │  │  (replica)   │  │              │  │  (primary)   │                 │
  │  └──────────────┘  └──────────────┘  └──────────────┘                 │
  │                                                                         │
  │  If Worker 1 dies → Coordinator uses Shard 1 replica on Worker 2       │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘

  Setup:

  -- On coordinator, BEFORE creating distributed tables
  SET citus.shard_replication_factor = 2;

  -- Now create distributed table
  SELECT create_distributed_table('monitors', 'organization_id');

  -- Each shard will be stored on 2 different workers

  ---
  NixOS Config for Full HA Citus

  # infra/nixos/modules/services/citus-ha.nix
  { config, lib, pkgs, ... }:

  {
    # Patroni for coordinator HA
    services.patroni = {
      enable = true;
      postgresqlPackage = pkgs.postgresql_16.withPackages (ps: [ ps.citus ]);

      settings = {
        scope = "citus-coordinator";

        postgresql = {
          parameters = {
            shared_preload_libraries = "citus";
            max_connections = 300;
          };
        };

        # Citus-specific: sync metadata on failover
        bootstrap.dcs.postgresql.parameters = {
          "citus.metadata_sync_interval" = 1000;
        };
      };
    };

    # PostgreSQL with Citus for worker role
    services.postgresql = {
      enable = true;
      package = pkgs.postgresql_16;
      extensions = ps: [ ps.citus ];

      settings = {
        shared_preload_libraries = "citus";
        listen_addresses = "*";
      };
    };
  }

  ---
  HA Comparison

  | Component   | Without HA                 | With HA         | Failover Time      |
  |-------------|----------------------------|-----------------|--------------------|
  | Coordinator | Single point of failure    | Patroni standby | ~10-30 sec         |
  | Workers     | Data loss if worker dies   | Shard replicas  | ~0 sec (automatic) |
  | etcd        | Cluster can't elect leader | 3-node quorum   | ~5 sec             |

  ---
  Minimum Nodes for Full HA

  | HA Level       | Coordinators | Workers | etcd | Total VPS |
  |----------------|--------------|---------|------|-----------|
  | No HA          | 1            | 1+      | 0    | 1         |
  | Worker HA only | 1            | 3+      | 0    | 3         |
  | Full HA        | 2            | 3+      | 3    | 3*        |

  *With dual-role nodes (coordinator+worker+etcd on same VPS)

CITUS HA


⏺ It's 2 levels that work together - you need both for full HA:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                         2 LEVELS = FULL HA                             │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  Level 1: Coordinator HA (Patroni)                                     │
  │  └─ Protects: Query routing, metadata, cluster management              │
  │                                                                         │
  │  Level 2: Worker HA (Shard Replication)                                │
  │  └─ Protects: Your actual data                                         │
  │                                                                         │
  │  You need BOTH for full HA                                             │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘

  What Happens If You Only Have One Level

  | Failure          | Only Coordinator HA   | Only Worker HA      | Both (Full HA) |
  |------------------|-----------------------|---------------------|----------------|
  | Coordinator dies | ✅ Failover to standby | ❌ Cluster down      | ✅ Works        |
  | Worker dies      | ❌ Data loss           | ✅ Use shard replica | ✅ Works        |

  Simple Answer

  Full HA = Coordinator HA + Worker HA

  Both are enabled independently:
  ├─ Coordinator HA → Enable Patroni
  ├─ Worker HA → Set shard_replication_factor = 2
  └─ They work together, not either/or

  With your 3 Netcup servers, you get both levels at no extra cost.
