  Now (222 writes/sec)
  └─ 2+1 is perfect

  └─ Upgrade primary: 4 CPU → 8 CPU → 16 CPU

  50x growth (11,000 writes/sec)

  100x growth (22,000 writes/sec)

  └─ Consider Citus sharding

  - ✅ Write scaling via vertical (upgrade primary)

  You won't need write sharding until 100x+ current scale.

  Summary

  | Question                | Answer                                    |
  |-------------------------|-------------------------------------------|
  | Does Oban add overhead? | Yes, ~5-10ms per job (DB operations)      |
  | Is it significant?      | No for most cases                         |
  | When to avoid Oban?     | High-frequency fire-and-forget jobs       |
  | When Oban is essential? | Jobs that MUST complete (alerts, billing) |

  Recommendation: Use hybrid approach:
  - Native OTP for monitor checking (high frequency)
  - Oban for notifications/webhooks (must retry)

  This gives you the best of both worlds.
